# Directorio: Historias de Usuario (User Histories - UHs)

## 📁 Propósito de la Carpeta
Actúa como el repositorio del **Product Backlog** del Módulo MR. Aquí se desglosa el trabajo en unidades ágiles orientadas al desarrollo de la base de datos y endpoints por sprints.

## 📄 Documentos que deben incluirse aquí:
- **`backlog_maestro.md`**: Lista priorizada de todas las Historias de Usuario divididas por Épicas (Proveedores, Finanzas, Infraestructura, Almacén, Seguridad).
- **`sprints/`**: Subcarpeta para archivar las historias asignadas a ciclos específicos (ej. `sprint_01_db_proveedores.md`).
- **`definition_of_done.md`**: Criterios obligatorios para dar por terminada una historia (pruebas unitarias, revisión de peers).

# Directorio: Requisitos Funcionales (Functional Requirements - FRs)

## 📁 Propósito de la Carpeta
Es el núcleo técnico de la funcionalidad. Aquí se almacena la ingeniería de detalle que los desarrolladores Backend y de Base de Datos (Derek, Mauricio, Erick, José, Diego) necesitan para estructurar el sistema relacional.

## 📄 Documentos que deben incluirse aquí:
- **`modelo_entidad_relacion.md`**: Explicación de las tablas (Proveedores, Finanzas, Espacios, Inventario, Accesos).
- **`diccionario_datos_mr.md`**: Especificación de tipos de datos, llaves primarias, foráneas y restricciones de unicidad (como el RFC).
- **`triggers_y_procedimientos/`**: Scripts o lógicas para los triggers de seguridad (automatización de registros de acceso a áreas restringidas).
